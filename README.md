# 噴霧（スプレー）異常検知の最小PoC

製造業で広く使われる噴霧工程（塗装・コーティング・噴霧乾燥など）を対象に、
「噴霧がうまく出ているか」をカメラ映像から自動で見張る仕組みの**原理を実証**する最小コードです。

この背景にある考え方は、記事に書いています。
（※note記事のリンクは公開後にここへ追記）

## このコードが示すこと

現場では、噴霧の良し悪しを人が目視で確認していることが多い。
これを自動化しようとすると、「窓越しでよく見えない」「ダストが舞っていて形が捉えにくい」
という壁にぶつかる。

そこで発想を変え、**噴霧の「形」ではなく「明るさの時間変化」**を信号として使う。
噴いていれば明るく、止まれば暗い。この明暗の変動なら、視界が悪くても比較的残りやすい。

このコードは、以下の3つの状態を合成データで見分けられるかを確認する：

- **正常**：明るさが高いまま安定
- **チラチラ（予兆）**：明るさが高速で上下にバタつく
- **詰まり**：明るさがゼロに張り付く

そして、**完全に詰まる前の「チラチラ」の段階で予兆を捉えられる**ことを示す。
手遅れのアラームではなく、予知につながる。

## 重要な前提

- **実データは一切使用していません。** すべてプログラムで生成した合成データです。
- これは「原理が成立するか」を確かめる第一歩であり、実機での動作を保証するものではありません。
- 実際の導入には、現場データでの検証や各種の確認が別途必要です。

## 実行方法

```
pip install numpy matplotlib
python spray_poc.py
```

実行すると、判定結果がコンソールに表示され、同じフォルダに2枚の図が出力されます。

- `poc_frames.png` … 窓越し・ダストを模した合成フレームの例（静止画では見分けにくい）
- `poc_result.png` … 明るさの時間変化で3つの状態が分離する様子

※ 日本語ラベルの表示には日本語フォント（Noto Sans CJK JP など）が必要です。
　 なくても動作しますが、図中の日本語が代替フォントになる場合があります。

## ファイル構成

- `spray_poc.py` … 本体（合成データ生成 → 明るさ抽出 → 判定 → 作図）
- `poc_frames.png` … 上記の出力例
- `poc_result.png` … 上記の出力例
